Prop

Prop is a binary proportional rhythmic notation system. With it’s simple syntax, prop can be used to notate complex rhythms.

Prop can be found on github here.

In prop, absolute time is arbitrary. Once a prop pattern has been analyzed, it is scaled based on the duration it is given.

Prop is a very simple syntax with only a few rules:

  • a ‘+’ is a note.
  • a ‘-‘ is a rest.
  • a positive integer N followed by parenthesis (ex: “2()”) will modify the time scale of a prop pattern inside the parenthesis by a factor of N. For instance, {2(++)} will produce a phrase twice as fast as {++}, 3(++) will produce a phrase three times as fast as {++}, etc, etc.
  • Parentheses can be encapsulated within eachother (ex: “{+2(+3(++))}”)
  • The entire prop pattern must be encapsulated inside brackets
  • Prop is whitespace insensitive.

CLI

The command-line interface, called “prp”, behaves very similarly to it’s sister program NRT in that it is a text filter: prop patterns go in, text tables come out.

Prp has a text table of two values, the first being the start time of the note, and the second value being the duration. The main idea behind this is that Prp can be chained together with other Unix programs to create music.

In addition to creating text tables, Prop has the addtional option of generating a visual representation of the prop pattern in postscript format.

Here are the possible flags to set:

-d time sets the duration (in seconds) of the prop pattern -p file will write a visual representation in postscript format to file
